🧠 思维理念
元原则 —— 科研的底层循环 ▼

先说预期 → 做实验/计算/阅读 → 解释实际与预期的差距

所有框架的底层是这一个循环。不管是 DFT 计算、读文献还是分析数据，这三个步骤反复执行。写下预期，才能知道自己"对"或"错"在哪里。

例：运行 Wien2k SCF 前写"预期自洽收敛在 50 步以内"，实际跑了 120 步——差距迫使你去查参数，而不是被动等待。

每日新鲜感与研究伙伴 ▼

今日研究微挑战：每天自动出现一个小的科研动作，例如写一句假设、记录异常、设计最小复现、补文献坐标。未完成前可以“换一个”；误触完成后可在当天撤销。

研究伙伴：可选择光谱猫、晶体狐、轨道龙或参数机器人。它会根据当天晨间目标、任务完成、晚间复盘和微挑战完成情况获得成长值，等级提升后形态变化。

时段话语：伙伴会根据早晨、中午、下午、晚上、深夜以及特定日期给出不同提醒。它的定位是陪你保持科研节奏，不替代真实记录。

升级阶梯详解（6 步） ▼
  1. 把问题写成一句话 —— 问题说不清楚，答案不可能清楚。例："费米面第3条能带在 Γ 点附近断裂，原因未知"
  2. 看极限情况 —— U→0, T→0 有没有解析解，或者最简单结构下会怎样
  3. 找类比 —— 类似体系（如同族化合物）的文献怎么处理这个问题
  4. arXiv / Google Scholar 搜索关键词 —— 先检索，避免重复踩坑
  5. 最小可验证计算 —— 用更小的 k-mesh 或更简单参数快速验证方向
  6. 带着具体问题去问人 —— AI / 同学 / 导师，但需带上前 5 步的结果
规则：不要跳步。不会怎么做 ≠ 可以直接跳到第 6 步去问导师。跳步的代价是问出一个模糊问题，得到一个模糊答案。

☁️ 云端同步说明 ▼

支持 Google Drive 和 GitHub 双端同步，任意一端配置即可跨设备使用（手机看、电脑改）。

Google Drive：点击顶部 按钮，用 Google 账号授权即可。数据存储在 Drive 应用隐藏目录，不占主目录空间。

GitHub：点击顶部 按钮，填写仓库名（用户名/仓库名）和 Personal Access Token（需有 Contents 读写权限）。

同步机制：修改内容 2 秒后自动推送；切回页面时自动拉取；每 30 秒轮询一次。两端同时配置时均生效，互为备份。手动点击 🔄 或按 Ctrl+S 可立即触发同步。

冲突处理：当云端数据比本地新时，会弹出本地/云端摘要，可选择“合并新增”“使用云端覆盖”或“保留本地并推送”。合并只导入本地不存在的日期/周计划/小记，同一天两端都改过时保留本地。

导出格式说明：

  • 预览：先查看导出卡片内容和公式渲染，再决定导出格式
  • PNG：高清图片，自动匹配当前深浅主题，支持 LaTeX 公式渲染
  • PDF：新窗口打开后自动弹打印对话框，选"另存为 PDF"即可
  • Markdown：纯文本，可粘贴到 Notion / Obsidian 等工具
  • JSON：完整数据备份，用于导入恢复

随手小记导出：选择"全部随手小记"时，会弹出选择框，可勾选需要导出的小记条目，支持全选/取消全选。

最佳实践：配置 GitHub 或 Drive 同步，数据自动备份到云端；重大编辑和云端恢复前会保留最近 5 次本地快照，同时定期用 💾 导出 JSON 作为离线备份。

恢复向导与冲突处理 ▼

导入恢复：导入 JSON 后先进入恢复向导，可按日期、周计划、小记和设置选择范围。日期支持全选、清空、反选、仅新增和跳过覆盖；部分导入会以合并方式写入，避免删除未选择内容。

冲突详情：同一天两端都修改时会保留本地，并显示字段级差异。可逐字段应用云端、批量使用云端、仅补云端新增字段或全部忽略。

科研建议：不确定时先导出完整 JSON，再做恢复或批量冲突处理。数据本身就是研究记录的一部分，保留可追溯性比追求一次性整理干净更重要。

故障排查 ▼
现象处理方式
同步失败打开“同步详情 / 自检”，查看 Drive 或 GitHub 最近状态；GitHub 常见原因是 Token 失效或仓库权限不足。
多端数据不一致出现冲突弹窗时先看本地/云端摘要；一般优先选“合并新增”，不确定时选择“稍后处理”，再导出 JSON 备份。
导入后担心覆盖错导入前会自动生成快照；可在“最近本地快照”中全部恢复，或只恢复某一天。
打卡没有续接检查上一天任务是否选择“当日完成”；如果是在历史编辑里补改昨天，保存后会自动补续接到今天。
导出图片不对先点“预览”确认公式和卡片内容；历史单日导出需要先在历史页打开某天详情。
